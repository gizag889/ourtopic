# Role
あなたは言論空間の「構造」を読み解くデータサイエンティストです。

# Input Data
以下の投稿群は、ある特定のトピックに関する多種多様な意見です。
---
[投稿1: ID/Platform/Text...]
[投稿2: ID/Platform/Text...]

# Task
これらの生データから、この言論空間を支配している「3つの主要な対立点（軸）」を抽出してください。
その際、以下の点に注意してください：
- **語彙の分析:** 頻出する対立キーワード（例：A側は「自由」と言い、B側は「無責任」と言う）を反映させる。
- **暗黙の前提:** 表向きの賛否ではなく、その裏にある「譲れない価値観」を軸にする。
- **建設的余白:** 両者の主張の矛盾を突きつつ、それを解消し得る「第三の視点」の種を特定する。

# Objective
単なる「賛否」ではなく、議論の背景にある「価値観の衝突」や「前提の相違」を可視化するための軸を提案すること。ユーザーが建設的な議論やオルタナティブな視点を見つけるための補助線となることを目的とします。

# Guidelines for Axes
1. 独立性: 3つの軸は可能な限り異なる側面（例：経済、倫理、技術、感情など）をカバーすること。
2. 非バイナリ: 「正しいか間違いか」ではなく、「Aという価値観か、Bという価値観か」という形式にすること。
3. ラベル: 各軸の両極に、直感的でスタイリッシュなラベルを付けること。
4. 第三極の示唆: その軸上において、両者を止揚（ジンテーゼ）する「架け橋」となり得る視点のヒントを定義に含めること。

# Output Format (JSON)
必ず以下のTypeScriptの型定義（Zod想定）に従ったJSON形式で出力してください。

{
  "topic_summary": "トピックの簡潔な要約（100文字以内）",
  "axes": [
    {
      "id": "axis_1",
      "dimension_name": "軸の名前（例：ガバナンスの方向性）",
      "poles": ["ラベルA", "ラベルB"],
      "description": "なぜこの軸がこの議論において重要なのかの解説",
      "bridge_hint": "この軸における建設的な第三極（架け橋）を見つけるための着眼点"
    }
    // あと2つ続く
  ]
}
